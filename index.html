2018年度总结

Veröffentlicht am 2019-01-14

    从去年9月入职至今，已4月有余。在此期间，我担任公司研发部的一名软件工程师，主要从事着JAVA项目的工作开发。
    入职以来，完成了对公司产品的梳理（一指蟾）和公司平台架构的学习；并在龙大佬的带领下配合其他小伙伴完成了一指蟾代理商H5和代理商移动端的开发工作，并顺利在上月初实现代理商H5上线，本月初实现代理商移动端上线。通过此次项目开发，加深了对一指蟾产品的理解，熟悉电商系统的业务流程。除此之外还对后端系统的技术架构有更进一步的了解，不论是spring cloud技术架构的实践应用，还是java基础编码能力提升，以及对云服务器结合生产环境的实际使用等等各方面，都有长足的进步和认识。
    在这一年里，不论是上半年的在前一家公司做的产品上线，还是在掌寻做的一指蟾产品上线；每一次项目开发过程中，都会遇到困难和阻碍，庆幸的是都解决了难题，正是因为一次次在困难中寻找解办法，在难题中寻找答案，才会让自己不断进步。毕业一年有余，从业两年有余，我深知自己进步不小，但同时差距也不小。在这瞬息变化的时代中，只有不断保持学习，提高自己的核心力，才不会掉队。一方面探索技术的广度，一方面深挖技术的深度。
    在一年里，自己也在尝试带领其他小伙伴一同进行项目开发。在技术上也能微末的为其他小伙伴提供一些帮助，在产品逻辑梳理上小伙伴们又会带给我一些启发，从而形成了相互促进的良好氛围。然而在团队协作上，却是有不足的。沟通不及时不到位，表述不清楚不透彻，导致我们做了一些无用功。这些问题都需要在新的一年里不断改进和完善，提高自己的团队合作意识。
    在一年里，自己也经历了人生第一次跳槽。世界那么大总想去看看，不安分的心永远在躁动。正是因为这颗不安分的心，才驱动着自己不断的学习进步。在自己所感兴趣的行业里，做一些自己真正觉得有趣的事情。正是因为不安分的心，才来到掌寻并结识这么多有趣的小伙伴，工作生活中才会充满乐趣。
    在新的一年里。技术上，脚踏实地，仰望星空。工作上，认真仔细，少出BUG！生活上，作息规律，绝不秃顶。学习上，完成2018年的书单目录！

No pain No gain.

加载原则：
Springboot的设计是加载应用级配置，随后再考虑自动配置类。比如已经配置了一个jdbcTemplateBean，那么在执行自动配置是就已经存在一个jdbcTemplate类型的Bean了，于是忽略默认的自动配置jdbcTemplateBean。

二、通过文件属性外置配置，按优先级排序

SpringBoot应用程序有多种设置途径，如：
1.命令行参数
2.java：comp/env里的JNDI属性
3.JVM系统属性
4.操作系统环境变量
5.随机生成的带random.*前缀的属性(在设置其他属性时，可以引用他们，比如${random.long})
6.应用程序以外的application.properties或者application.yml文件
7.打包在应用程序内的application.properties或者application.yml文件
8.通过@PropertySource标注的属性
9.默认属性
其中：application.properties和application.yml能放在以下四个位置，按优先级排序
1.外置。在相对于应用程序运行目录的/config子目录里
2.外置。在应用程序运行的目录里
3.内置。在config包内
4.内置。在Classpath根目录
注意:在同一优先级位置有application.properties和application.yml两个文件时，那么application.yml的属性会覆盖application.properties。

运用Spring Boot应用

使用上述命令快速构建一个SpringBoot应用
ps：注意CLI的init命令是不能指定项目根包名和项目名，需要在打开项目后将包名和启动类修改。

启动引导类

@SpringBootApplication:开启组件扫描和自动配置
等同于以下三个注解：
@Configuration:表明该类使用Spring基于java的配置
@ComponentScan:启用组件扫描
@EnableAutoConfiguration:开启SpringBoot自动配置的关键注解

运行SpringBoot

  1. 用gradle的bootRun任务

    gradle bootRun

  2. 先用gradle构建项目，然后在命令行里用java运行

    gradle build
    java -jar build/libs/xxx.jar

其他注解

@ConditionalOnBean:配置了某个特定的Bean
@ConditionalOnMissingBean：没有配置特定的Bean
@ConditionalOnClass:Classpath里有指定的类
@ConditionalOnMissingClass：Classpath中缺少指定的类
@ConditionalOnExpression：给的Spring Expression Language表达式结果为true
@ConditionalOnJava：Java的版本匹配特定值或者一个范围值
@ConditionalOnJndi：参数中给定的JNDI位置必须存在一个，如果没有参数，则要有JNDI InitialContext
@ConditionalOnProperty：指定的配置属性要有一个明确的值
@ConditionalOnResource：Classpath里有指定的资源
@ConditionalOnWebApplication：这是一个web应用程序
@ConditionalOnNotWebApplication：这不是一个web程序

SpringBoot读书笔记-第一节

Veröffentlicht am 2018-11-29 | Edited on 2019-01-11

Spring架构

core container(核心容器)

Spring-core:核心工具类
Spring-Beans:Spring定义的Bean支持
Spring-Context:运行时的Spring容器
Spring-Context-Support:Spring容器对第三方的继承支持
Spring-Expression:使用表达式语言在运行时查询和操作对象

AOP(面向切面编程)

Spring-AOP:基于代理的AOP支持
Spring-Aspects:基于Aspects的AOP支持

Messaging(消息)

Spring-Messaging:对消息架构和协议的支持

Web

Spring-Web:提供基础的Web继承功能，在Web项目中提供Spring的容器
Spring-WebMVC:提供基于Servlet的SpringMVC
Spring-WebSocket:提供WebSocket功能
Spring-WebMVC-Portlet:提供Portlet环境支持

Data Access/Integration

Spring-JDBC:提供以JDBC访问数据库的支持
Spring-TX:提供编程式子和申明式的事物支持
Spring-ORM:提供对对象/关系映射技术的支持
Spring-OXM:提供对对象/xml映射技术的支持

Spring-JMS:提供对JMS的支持

Spring生态

SpringBoot:快速开发的脚手架
Spring XD:简化大数据应用的开发
Spring Cloud:分布式系统开发工具类
Spring Data:对主流关系型和NoSQL数据库的支持
Spring Integration:通过消息机制对企业集成模式的支持
Spring Batch:简化及优化大量数据的批处理操作
Spring Security:通过认证和授权保护应用
Spring HATEOAS:基于HATEOAS原则简化REST服务开发
Spring Social:与社交网络API的集成
Spring AMQP:对基于AMQP的消息支持
Spring Mobile:提供对手机设备检测的功能
Spring for Android:主要提供在Android上消费RESTFul API的功能
Spring Web Service:提供了基于优先的SOAP/Web服务
Spring LDAP:简化使用LDAP开发
Spring Session:提供一个API及实现来管理用户的会话信息

Spring 四大原则

  1. 使用pojo(plain old object)进行轻量级和最小级侵入开发
  2. 通过依赖注入和基于接口编程实现松耦合
  3. 通过AOP和默认习惯进行申明式编程
  4. 通过AOP和模板减少模式化代码

SpringBoot精要-四个核心

  1. 自动配置：针对常见Spring应用程序功能自动提供相关配置
  2. 起步依赖：只需关注什么功能，SpringBoot会根据依赖导入相应的依赖包
  3. 命令行界面：可选特性，让只写代码即可实现应用程序成为可能
  4. Actuator：深入了解运行的SpringBoot程序
    作用：
    a.Spring应用程序上下文配置里Bean
    b.Springboot的自动配置做的决策
    c.应用程序取到的环境变量、系统属性、配置属性和命令行参数
    d.应用程序里线程的当前状态
    e.应用程序最近处理过的HTTP请求的追踪情况
    f.各种内存用量、垃圾回收、Web请求以及数据源用量相关的指标
